# 13 — Glossary

Terms used throughout this reference, in alphabetical order.

---

**Action** — A user-triggered side effect described as an `ActionConfig` and dispatched through `useActionHandler`. Six types: `navigate`, `api-mutation`, `open-modal` / `open-sheet`, `api-download`, `trigger-event`, `update-widget-state`. See [05-actions.md](05-actions.md).

**ActionBar** — Widget (`type: "action-bar"`) that renders a row of buttons gated by entity state and user role. The standard pattern for entity-level workflow actions. See [02-widget-catalog.md → action-bar](02-widget-catalog.md#action-bar).

**ActionConfig** — TypeScript discriminated union describing any user action. Defined in [`src/types/widget.ts`](../../src/types/widget.ts).

**App context** — The metadata + navigation menu fetched once at startup from `/api/config/app`. Powered by `<AppContextProvider>`.

**Backend errors** — Field-level error envelope returned by mutations: `{ backendErrors: [{ variable_code, error_code, error_desc }] }`. Forms map `variable_code` to field `name` for inline display. See [06-forms.md → Backend errors](06-forms.md#backend-errors).

**`backendErrors` shape** — Legacy field-error envelope. New endpoints should return both this and the top-level `message` for compat.

**Cache invalidation** — Forcing TanStack Query to refetch. Triggered via `action.refreshKey` (prefix match) after a mutation succeeds.

**Catch-all proxy** — A Next.js dynamic route (`[[...path]]`) that forwards every request under its prefix to an external backend. The standard pattern for switching from mock routes to a real backend.

**`children`** — Field on `WidgetConfig`. Array of nested widget configs. Used by container widgets (`stack-layout`, `tabs-container`, `grid-layout`, …) to compose deeper structures.

**colSpan** — `WidgetConfig.layout.colSpan`. When the widget is a child of a `grid-layout`, this controls how many columns it spans (1–4).

**Confirmation dialog** — A modal that appears before an `api-mutation` fires, triggered by setting `confirm: { title, message }` on the action. Rendered by `<ConfirmationDialog />` via `useOverlayStore`. See [05-actions.md → Confirmation](05-actions.md#confirmation-dialogs).

**DataSource** — `WidgetConfig.dataSource`. Describes how to fetch data for a widget — endpoint, polling, state dependencies, value extraction. Consumed by `useSmartQuery`. See [04-data-sources.md](04-data-sources.md).

**DataSourceConfig** — TypeScript shape of `dataSource`. Defined in [`src/types/widget.ts`](../../src/types/widget.ts).

**Detail page** — A Next.js page under `src/app/<module>/[id]/page.tsx` that resolves a `<module>-detail.json` schema, substitutes the route param into endpoints, and renders.

**`disabledTooltip`** — Field on `BaseActionConfig`. When set, the action renders disabled with the tooltip shown on hover. Used for "this exists but isn't ready yet" UX where backend support is missing.

**`displayStyle`** — Field on form field configs. Controls how the field renders in view mode (e.g., `"text"`, `"date"`, `"currency"`). The edit mode always shows the input control.

**Endpoint substitution** — Replacing `:paramName` tokens in URL strings with values from `rowData` (or form values). Implemented by `substituteEndpointParams` in [`src/lib/endpointUtils.ts`](../../src/lib/endpointUtils.ts).

**Entity** — A backend resource (claim, policy, quote, …) that has a lifecycle state. The framework uses this loosely; state widgets like `state-badge` accept an `entity` prop with known options.

**`enhancedProps`** — Internal name for the merged props the `WidgetRenderer` passes to each component (original `props` + fetched `data` + `isLoading` + `error`).

**Field types** — The string values accepted by `FormFieldConfig.type`: `text`, `number`, `email`, `url`, `password`, `textarea`, `date`, `select`, `radio`, `checkbox`, `file`. See [06-forms.md → Field types](06-forms.md#field-types).

**`fields`** — On `form-container`, the array of field configs. On `key-value-grid` and `info-card`, also the array of field configs (different shape).

**Filter bar** — Widget (`type: "filter-bar"`, alias `search-bar`) that renders a search + filter UI. Publishes state to `useWidgetState`; paired with a `data-table` via `stateDependencies`. See [02-widget-catalog.md → filter-bar](02-widget-catalog.md#filter-bar--search-bar).

**Forms registry** — `schemas/forms/index.ts`. Auto-generated bundle of every form schema, keyed by form id. Generated by `scripts/generate_form_index.mjs`, runs on `npm run predev`.

**FormFieldConfig** — The shape of a single field in `form-container.props.fields`. See [06-forms.md → Common field props](06-forms.md#common-field-props).

**`global:current-role`** — The `useWidgetState` key under which `RoleProvider` publishes the active role. Conditions can read it via `{ "var": "global:current-role" }`.

**`hasWorkflow`** — `tabs-container` prop. When true, the container shows Prev / Next / Complete buttons and emits workflow events.

**`headerActions`** — Toolbar buttons on a `data-table`. Distinct from `rowActions` (per-row) and `bulkActions` (selected-rows-only).

**`id`** — Required field on every `WidgetConfig`. Unique within the page. Also the form-registry key for form schemas.

**`info-card`** — Feature-branch widget that overlaps with `key-value-grid`. Not in the main registry on `main`. See [02-widget-catalog.md → Feature-branch widgets](02-widget-catalog.md#feature-branch-widgets--not-on-main).

**JSONLogic** — Declarative predicate language used for `visibleWhen` and `stopWhen`. Implemented via [json-logic-js](https://jsonlogic.com). See [07-state-and-conditions.md → JSONLogic](07-state-and-conditions.md#jsonlogic).

**`layout`** — `WidgetConfig.layout`. Rendering hints — `colSpan`, `hidden`, `visibleWhen`.

**List page** — A Next.js page (`src/app/<module>/page.tsx`) that renders a module's `<module>.json` schema, usually a header + filter-bar + data-table.

**Lucide** — The icon library used everywhere. Icons are referenced by string name (`"Eye"`, `"Edit"`, …). See [lucide.dev](https://lucide.dev).

**Mock route** — A Next.js route handler under `src/app/api/...` that returns fixture data for local dev. Replaced by proxy routes in QA/prod.

**Mode (view / edit)** — `form-container.props.mode`. `"view"` renders the form read-only; `"edit"` (default) renders inputs.

**Mutation** — A backend write action — POST, PUT, PATCH, DELETE. Dispatched in this framework via an action with `type: "api-mutation"`. See [05-actions.md → api-mutation](05-actions.md#api-mutation).

**`api-mutation`** — `ActionConfig` variant that fetches an endpoint (POST/PUT/PATCH/DELETE/GET-for-refresh), shows a toast on success, invalidates caches via `refreshKey`, and dispatches `onSuccess` chains. The standard write-action shape.

**`navigate`** — `ActionConfig` variant that calls `router.push(target)`. Supports `:param` substitution from row data.

**`open-modal` / `open-sheet`** — `ActionConfig` variants that open an overlay via `useOverlayStore`. `target` is typically a form id from the forms registry.

**`api-download`** — `ActionConfig` variant that fetches the endpoint as a blob and triggers a browser download.

**`trigger-event`** — `ActionConfig` variant that emits a host event. On `main` this primarily means "close an overlay" — `target` is the overlay id.

**`update-widget-state`** — `ActionConfig` variant that mutates `useWidgetState` directly. Operations: `set`, `patch`, `toggle`.

**`submitAction`** — A boolean flag on a form action. The action marked `submitAction: true` is the one that fires on Enter/Submit. The form's field values become the `api.body` automatically when the action is `api-mutation`. See [06-forms.md → Form actions](06-forms.md#form-actions).

**OverlaidForm** — Component (`src/components/widgets/forms/OverlaidForm.tsx`) that renders a form inside a modal/sheet. Triggered by `open-modal` / `open-sheet` actions.

**Overlay** — A modal, sheet, or dialog. Managed via `useOverlayStore`. Mounted globally by `<OverlayProvider />`.

**Page-level template substitution** — Server-side string replacement of `{{id}}` in the resolved schema, performed by the page's `updateEndpoints` walker. See [08-pages-and-routing.md → Template substitution](08-pages-and-routing.md#template-substitution).

**`pollSchedule`** — `DataSourceConfig.pollSchedule`. Tiered backoff polling: fast initially, then slower, with a hard max duration. See [04-data-sources.md → Polling](04-data-sources.md#polling).

**Primitive** — A base UI component in `src/components/ui/*` (Button, Card, Badge, …). Widgets are *composed of* primitives. Don't reach for primitives in schemas — use widgets.

**Provider** — A React Context provider in the app's provider stack. Examples: `QueryClientProvider`, `RoleProvider`, `OverlayProvider`. See [01-architecture.md → The provider stack](01-architecture.md#the-provider-stack).

**Proxy mode** — Running the framework with the catch-all proxy route forwarding to a real backend, instead of using mock routes.

**`refreshInterval`** — `DataSourceConfig.refreshInterval`. Fixed-interval polling in milliseconds. See [04-data-sources.md → Polling](04-data-sources.md#polling).

**`refreshKey`** — `BaseActionConfig.refreshKey`. Prefix string. After a successful mutation, every TanStack Query whose key starts with this prefix is invalidated.

**`$ref`** — Schema composition shortcut. `{ "$ref": "schemas/tabs/<...>.json" }` loads and splices that file's content at the ref node. See [03-schemas.md → $ref](03-schemas.md#ref--composing-schemas).

**Role** — A user permission identifier (`claims_adjuster`, `claims_supervisor`, `maker`, `checker`, …). Used to gate widget visibility and action availability.

**`roleActions`** — `action-bar` prop. Map of role → array of action IDs visible to that role.

**`rowActions`** — `data-table` prop. Per-row action menu (typically a kebab).

**`rowData`** — The data of a specific row, passed to `useActionHandler.dispatch` for row actions. Used for `:param` substitution and modal data prefill.

**`rowIdKey`** — `data-table` prop. The field on row data that serves as the PK. Default `"id"`. Override for non-id PKs.

**Schema** — A JSON file describing a screen via `WidgetConfig` tree. Lives under `schemas/`.

**Schema resolver** — `src/lib/schemaResolver.ts`. Implements `resolveSchemaRefs` — recursive `$ref` expansion.

**`section-group`** — Layout widget for titled grid sections. See [02-widget-catalog.md → section-group](02-widget-catalog.md#section-group).

**Server component** — A Next.js page or component that runs on the server, async-capable, no hooks. Pages are server components by default. Widgets are *not*.

**State (entity)** — The lifecycle status of an entity (claim state `TRIAGED`, policy state `ACTIVE`). Used by `action-bar` to gate actions.

**State (UI shared)** — Cross-widget state stored in `useWidgetState`. Filter values, form drafts, modal payloads.

**`stateActions`** — `action-bar` prop. Map of state → array of action IDs allowed in that state.

**`stateDependencies`** — `DataSourceConfig.stateDependencies`. Array of `useWidgetState` keys that, when changed, trigger refetch and contribute to the query cache key.

**`stateField`** — `action-bar` prop. Field name on the fetched entity that holds the state. Default `"state"`; Quote uses `"status"`.

**`stateKey`** — Various widgets accept this prop to declare where in `useWidgetState` they publish their state. Conventions: `page:<module>:filters`, `wizard:<id>:step`, etc.

**`stopWhen`** — `DataSourceConfig.stopWhen`. JSONLogic predicate that halts polling early when truthy.

**`submitAction`** — `form-container.props.actions[].submitAction`. Marks the action that fires on Enter/Submit. Exactly one action should have this flag.

**`substituteEndpointParams`** — `src/lib/endpointUtils.ts`. Replaces `:paramName` tokens in URLs with values from `rowData`.

**`successMessage`** — `api-mutation` action prop. The toast text on success. **Always set this** — silent submits are confusing.

**Tab panel** — A widget config with `type: "tab-panel"` used as a child of `tabs-container`. Has `label`, `icon`, and `children`.

**Tabs container** — Widget (`type: "tabs-container"`) for tabbed content. See [02-widget-catalog.md → tabs-container](02-widget-catalog.md#tabs-container).

**Template substitution** — See *Endpoint substitution* and *Page-level template substitution*. Two distinct mechanisms (`:param` vs `{{id}}`); don't confuse them.

**Token (design)** — A CSS custom property in `globals.css` representing a brand value (colour, radius, spacing). Used via Tailwind utility classes (`bg-primary`, `rounded-lg`).

**Token (route)** — A parameter placeholder in a URL string (`:id`, `:claim_id`). Substituted at action-dispatch time.

**`trigger-event`** — Action type that emits a host event. Today, primarily used to close overlays (`target` is the overlay id).

**`update-widget-state`** — Action type that mutates `useWidgetState` directly. Operations: `set`, `patch`, `toggle`.

**`useActionHandler`** — Hook that dispatches any `ActionConfig`. Lives in [`src/hooks/useActionHandler.ts`](../../src/hooks/useActionHandler.ts). See [05-actions.md](05-actions.md).

**`useDataTable`** — Hook used inside `<DataTable />`. Wraps TanStack Table — sorting, filtering, pagination, selection.

**`useFormContainer`** — Hook used inside `<FormContainer />`. Wraps react-hook-form + Zod, applies field-level visibility filtering.

**`useOverlayStore`** — Zustand store managing open modals/sheets/dialogs. See [05-actions.md → Modals](05-actions.md#open-modal--open-sheet).

**`useRole`** — Hook returning `{ role, setRole }` from `RoleContext`. The role is also published to `useWidgetState["global:current-role"]`.

**`useSmartQuery`** — Hook used by `WidgetRenderer` to execute `dataSource`. Wraps TanStack Query with state-aware params, polling, and JSONLogic stop conditions. See [04-data-sources.md](04-data-sources.md).

**`useTableExport`** — Hook used by `<DataTable />` for client-side CSV / XLSX / PDF export.

**`useWidgetState`** — Zustand store for cross-widget shared state. See [07-state-and-conditions.md → useWidgetState](07-state-and-conditions.md#usewidgetstate).

**Validation rule** — An entry in `field.validations`: `{ rule, value?, message }`. Supported rules: `required`, `min`, `max`, `pattern`, `email`, `url`.

**`valueKey`** — `DataSourceConfig.valueKey`. Dotted path to extract from the response (e.g., `"items"` from `{ items: [...] }`).

**`valueMapping`** — Array on column configs (data-table) and field configs (key-value-grid, info-card) defining `{ value, label, color?/variant? }` mappings. Used for badge rendering.

**Variant** — A CVA-defined visual style for a primitive (button variants: `default`, `outline`, `destructive`, …). Schemas reference these by name via `action.variant` or column type.

**`visibleWhen`** — JSONLogic predicate for conditional visibility. Available on form fields (visibleWhen against form values) and widget `layout` (visibleWhen against fetched data + role).

**Widget** — A registered component identified by a `type` string in the registry. The unit of composition in schemas.

**WidgetConfig** — The TypeScript type representing one node in a schema tree. Defined in [`src/types/widget.ts`](../../src/types/widget.ts).

**Widget registry** — `src/components/registry/WidgetRegistry.tsx`. The `type` → component map.

**WidgetRenderer** — `src/components/registry/WidgetRenderer.tsx`. The recursive component that consumes a `WidgetConfig`, fetches its dataSource, looks up the component, and renders.
