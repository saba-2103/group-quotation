  # Keystone UI — Component Library

  This is the Storybook for **Keystone UI**, a schema-driven UI platform built on Next.js and deployed on Cloudflare Workers via OpenNext.

  > **Core Philosophy:** _"Everything is a Widget."_ Pages are not hardcoded — they are JSON schemas rendered recursively by the `WidgetRenderer` engine.

## Tech Stack

<div className="stack">
  <div className="stack-item"><strong>Framework</strong> Next.js 16 (App Router) on Cloudflare OpenNext</div>
  <div className="stack-item"><strong>UI Components</strong> Shadcn UI + Radix UI primitives</div>
  <div className="stack-item"><strong>Styling</strong> Tailwind CSS v4 with <code>class-variance-authority</code> (CVA)</div>
  <div className="stack-item"><strong>State</strong> Zustand (global) + TanStack Query (server state)</div>
  <div className="stack-item"><strong>Forms</strong> React Hook Form + Zod validation</div>
  <div className="stack-item"><strong>Icons</strong> Lucide React</div>
  <div className="stack-item"><strong>Schema Storage</strong> Cloudflare KV (edge-local reads)</div>
</div>

## Stories Structure

Stories are **centralized** in `src/stories/` and mirror the component structure.

<pre>
{`src/stories/
└── ui/                         ← Shadcn base primitives
    ├── button.stories.tsx      → Button (6 variants, 8 sizes)
    ├── badge.stories.tsx       → Badge (7 variants incl. success/warning/info)
    ├── input.stories.tsx       → Input (default, disabled, error states)
    ├── card.stories.tsx        → Card (with header, footer, metric layout)
    ├── skeleton.stories.tsx    → Skeleton (line, paragraph, card, table)
    └── tabs.stories.tsx        → Tabs (default pill + case underline variants)`}
</pre>

## Component Categories

<div className="grid">
  <div className="card">
    <h3>🧱 UI Primitives</h3>
    <p>Base Shadcn components — Button, Badge, Input, Card, Skeleton, Tabs. These are the lowest-level building blocks, fully styled with Tailwind CVA variants.</p>
  </div>
  <div className="card">
    <h3>🪟 Layout Widgets</h3>
    <p>Schema-driven layout containers — <code>stack-layout</code>, <code>grid-layout</code>, <code>tabs-container</code>, <code>section-group</code>. These are resolved by the <code>WidgetRenderer</code>.</p>
  </div>
  <div className="card">
    <h3>📊 Data Widgets</h3>
    <p>Smart data-fetching widgets — <code>data-table</code>, <code>key-value-grid</code>, <code>metric-card</code>. They use <code>useSmartQuery</code> to bind to a <code>dataSource</code> in the schema.</p>
  </div>
  <div className="card">
    <h3>📝 Form Widgets</h3>
    <p>Schema-driven forms — <code>form-container</code>, <code>filter-bar</code>. Fields and validations are defined in JSON and rendered dynamically with React Hook Form + Zod.</p>
  </div>
</div>

## How to Add a New Story

Place new story files inside `src/stories/ui/` (for primitives) and follow this pattern:

<pre>
{`import type { Meta, StoryObj } from '@storybook/react';
import { YourComponent } from '../../components/ui/your-component';

const meta: Meta<typeof YourComponent> = {
    title: 'UI/YourComponent',
    component: YourComponent,
    tags: ['autodocs'],
};

export default meta;
type Story = StoryObj<typeof YourComponent>;

export const Default: Story = {
    args: { /* your props here */ },
};`}
</pre>
